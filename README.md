# Безопасная обработка токенов и сессионных идентификаторов

## Функциональные требования
1. Перехват и анализ HTTP-трафика:
    Система должна обеспечивать анализ HTTP-запросов и ответов веб-приложения с целью выявления небезопасного использования токенов и сессионных идентификаторов.
    Функции:
    1) Перехват HTTP/HTTPS-трафика между клиентом и тестируемым веб-приложением;
    2) Извлечение из трафика:
      a.	cookie, 
      b.	заголовков Authorization, 
      c.	параметров URL, 
      d.	параметров тела запроса;
    3) Выявление токенов доступа и идентификаторов сессий. 
2. Анализ способов передачи токенов:
    Система должна анализировать способы передачи токенов и выявлять небезопасные практики.
    Функции:
    1) Определение передачи токена:
      a.	в cookie, 
      b.	в заголовках, 
      c.	в URL-параметрах, 
      d.	в теле запроса; 
    2) Обнаружение передачи токенов:
      a.	по незащищенному протоколу HTTP, 
      b.	в URL (риск утечки через историю браузера и логи); 
    3) Проверка атрибутов cookie:
      a.	Secure, 
      b.	HttpOnly, 
      c.	SameSite. 
3. Правило-ориентированный аудит безопасности:
    Система должна выполнять аудит использования токенов на основе набора правил безопасности.
    Функции:
    1) Проверка соответствия использования токенов набору правил безопасности;
    2) Поддержка конфигурации правил аудита;
    3) Выявление следующих нарушений:
      a.	токен передается в URL, 
      b.	токен передается по HTTP, 
      c.	cookie без атрибута Secure, 
      d.	cookie без HttpOnly,
      e.	отсутствие ограничений SameSite,
      f.	повторное использование одного и того же session id из разных IP/UA. 
4. Тестирование системы проведением атак:
    Тесты:
    1)	Проведение атаки session hijacking путем повторного использования перехваченного session ID; 
    2)	Проведение атаки session fixation путем использования заранее заданного session ID; 
    3)	Анализ реакции веб-приложения на такие атаки; 
    4)	Определение успешности атаки. 
5. Обнаружение подозрительного использования токенов:
    Система должна выявлять аномалии в использовании токенов.
    Функции:
    1)	Обнаружение использования одного токена: 
      a.	с разных IP-адресов,
      b.	с разных user-agent,
    2)	Обнаружение резкой смены географии или клиента; 
    3)	Обнаружение повторного использования истекших токенов. 
6. Формирование отчета безопасности:
    Система должна формировать отчет по результатам анализа.
    Функции:
    1)	Вывод списка обнаруженных уязвимостей; 
    2)	Классификация уязвимостей по типам атак: 
      a.	session hijacking, 
      b.	session fixation, 
      c.	утечка токенов; 
    3)	Рекомендации по устранению.
