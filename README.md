# Безопасная обработка токенов и сессионных идентификаторов

## Функциональные требования
1. Управление сессиями:
    - Создание сессии и выдача клиенту идентификатора сессии должны выполняться
    только после успешной аутентификации
    - Идентификатор сессии должен быть сгенерирован криптографически стойким
    ГПСЧ
    - При "выходе" пользователя должна выполняться инвалидация использованного
    идентификатора сессии
    - Поддержка продления сессии с помощью реаутентификации
    - Поддержка таймаутов сессий: таймаут бездействия и максимальный таймаут
    (максимальная длительность сессии вне зависимости от активности). Активность
    сбрасывает таймаут бездействия, а реаутентификация - оба
2. Выдача и область действия cookie:
    - Поддержка cookie только для защищенных сессий (HTTPS)
    - Помечать сессионные cookie как HttpOnly
3. Access tokens:
    - Прием токенов только в заголовках/теле
4. Безопасное хранение идентификатора сессии на клиентской стороне
5. Логирование и аудит:
    - Логирование событий: успешный/неуспешный вход в систему, выход из системы,
    создание/истечение/завершение сессий
    - Сокрытие идентификаторов сессий и токенов в логах
