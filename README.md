# Безопасная обработка токенов и сессионных идентификаторов

## Функциональные требования
1. Перехват и анализ HTTP-трафика. Система должна обеспечивать анализ HTTP-запросов и ответов веб-приложения с целью выявления небезопасного использования токенов и сессионных идентификаторов.
    - Перехват HTTP/HTTPS-трафика между клиентом и тестируемым веб-приложением;
    - Извлечение из трафика:
      -	cookie, 
      -	заголовков Authorization, 
      -	параметров URL, 
      -	параметров тела запроса;
    - Выявление токенов доступа и идентификаторов сессий. 
2. Анализ способов передачи токенов. Система должна анализировать способы передачи токенов и выявлять небезопасные практики.
    - Определение передачи токена:
      -	в cookie, 
      -	в заголовках, 
      -	в URL-параметрах, 
      -	в теле запроса; 
    - Обнаружение передачи токенов:
      -	по незащищенному протоколу HTTP, 
      -	в URL (риск утечки через историю браузера и логи); 
    - Проверка атрибутов cookie:
      -	Secure, 
      -	HttpOnly, 
      -	SameSite. 
3. Правило-ориентированный аудит безопасности. Система должна выполнять аудит использования токенов на основе набора правил безопасности.
    - Проверка соответствия использования токенов набору правил безопасности;
    - Поддержка конфигурации правил аудита;
    - Выявление следующих нарушений:
      -	токен передается в URL, 
      -	токен передается по HTTP, 
      -	cookie без атрибута Secure, 
      -	cookie без HttpOnly,
      -	отсутствие ограничений SameSite,
      -	повторное использование одного и того же session id из разных IP/UA. 
4. Тестирование системы проведением атак. Тесты:
    - Проведение атаки session hijacking путем повторного использования перехваченного session ID; 
    - Проведение атаки session fixation путем использования заранее заданного session ID; 
    - Анализ реакции веб-приложения на такие атаки; 
    - Определение успешности атаки. 
5. Обнаружение подозрительного использования токенов. Система должна выявлять аномалии в использовании токенов.
    - Обнаружение использования одного токена: 
      -	с разных IP-адресов,
      -	с разных user-agent,
    - Обнаружение резкой смены географии или клиента; 
    - Обнаружение повторного использования истекших токенов. 
6. Формирование отчета безопасности. Система должна формировать отчет по результатам анализа.
    - Вывод списка обнаруженных уязвимостей; 
    - Классификация уязвимостей по типам атак: 
      -	session hijacking, 
      -	session fixation, 
      -	утечка токенов; 
    - Рекомендации по устранению.
